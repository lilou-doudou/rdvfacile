# RDVFACILE V2 - Dossier de cadrage Mairie

Ce dossier regroupe les livrables de cadrage pour faire évoluer RDVFACILE vers une version adaptée aux mairies et aux services administratifs.

## Documents

- `01-plan-evolution-v2-mairie.md` : plan d'évolution produit et projet
- `02-cahier-fonctionnalites-prioritaires.md` : cahier des fonctionnalités prioritaires
- `03-architecture-donnees-v2-mairie.md` : proposition d'architecture de données cible

## Objectif

Faire passer RDVFACILE d'un produit de gestion de rendez-vous pour commerces à une plateforme de gestion de rendez-vous multicanal pour services publics, avec :

- gestion multi-services
- gestion multi-guichets et multi-agents
- règles métiers administratives
- traçabilité
- notifications
- portail citoyen

## Positionnement V2

RDVFACILE V2 doit pouvoir servir :

- une mairie
- une préfecture
- un centre administratif
- un établissement public

La priorité immédiate reste la mairie, avec une approche pilote sur un périmètre limité de services.
